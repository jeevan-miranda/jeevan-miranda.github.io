Blocked on a simple problem

What was the problem?

For the kata 5-Built-in methods, when doing the deBee.js question, I was able to build a function that removed the buzz word from the string regardless of capitalization by using the replaceAll() string method, however while the function console logged correctly, it did not pass the last test.

What problem-solving techniques did you use?

I first tried looking at the function in the test file and tried googling the methods used to understand what the function was trying to solve and how it worked, however I didn't understand how my function was still failing the test.
After about 30 min of trying to reverse engineer the test function to understand it, I decided to reach out to the help channel to get some idea of the problem.
Gerard was able to help me with understanding the big picture of the test and that the intention of the exercise was to solve it with arrays. This pointed me in the right direction.
I then had another student notice my question on the help channel and offered a hint to help me out. I was appreciative but wanted to figure it out on my own, but took the chance to rubber duck with them over discord, helping me put into words what I needed to achieve in plain english, as steps.
I then researched how to solve the issue using arrays, and was able to correct my function to solve the last test

How did you feel throughout the process?

I often feel nervous asking for help, but since learning about the benefits of maintaining a growth mindset, I've been more open to challenge that uncomfortable feeling I have about asking for help and get over it. Now that I've done it, I don't feel uncomfortable about it anymore.

What did you learn?

The main lesson I learnt is that there are multiple ways to approach a solution, but determining the right one depends on how well you understand the question.

Elegantly solved a problem

What was the problem?

I was trying to find a way to create a drop down paragraph that was revealed when clicking on the title header. When formulating my initial function, I ran into an issue where clicking the header for one para opened all the other hidden paragraphs too.

I wasn't able to solve the issue in my js file before the sprint 3 deadline, so I pseudocoded it to address later. I then came back during sprint 4 and did alot of googling to find different techniques people use to achieve an accordian style hidden paragraphs, then went back and unpseudocoded my function and refined it, before testing it out in the DOM to confirm it worked.

How did you feel throughout the process?

I felt pretty satisfied with my work, especially now that it is a re-useable peice of script I can use for the rest of my blogs

I learn't the importantance of making code more re-useable.
